### Azure active directory identity protection
**What identity protection is? and how it works?**
Azure AD Identity protetion is an identity admin tollbox to prevent, detect and reemdiate identity risk in their organisation. It monitors every login for identity compromise using numerous detection. 

These detection can be based on(Azure AD Identity Protection- Detections)
1) Heuristis
2) Machine learning 
3) Partner Products
4) These detections are supper valuable.
They are the secret treat of Microsoft source of identity protection.

Microsoft use these detections to calculate:
1) User risk: Probability an identity is compromised
2) Sign-in -risk: Probability a Sign-in is compromised------------Sign-in can be a **real time(based on real time detection)** or **aggregate(Based on all detections-rael time or Non real time)** type, including the ones that are issued after the sign in has been done. These risks arre shown as High, Medium or Low in identity proettcion.



**How Microsoft uses these detections to help customers**
Thse detections are like sunny days in seatle. THEY ARE VERY VAAUBALE. Customers can use these risks to set up automated remediation policies.
**Risk Policies include:**
1) User risk policy
2) Sign in risk policy

**These risk policies help oragnisations in multiple ways.**
1) They balance security and proudctivity: For example, all sign-ins are not challenged with an MFA, Hover if risk is detrected on a sign-in or user---microsoft will challenge the sign-ins based on the policies.
2) Reduced time to respond after a compromise:
3) Reduce help-desk cost: Once we setup the policies no manual intervention is required.
